## 代码规范 (Code Standards)
 
这些规则确保代码的一致性、可维护性和可扩展性。
**重要**: 这些规则不仅适用于重构代码，更是编写新代码时必须遵循的标准。

### 一、分层解耦原则

#### 1.1 核心层与UI层分离
- `core/` 模块不得直接返回面向用户的文本字符串。
- core 层统一返回 i18n 标签（`make_status_tag`），UI 层统一解析（`resolve_i18n_text`）。
- 示例文件：`agnets/example-04-core-i18n.md`

#### 1.2 业务逻辑与视图分离
- 提取UI相关代码到独立模块，保持核心模块纯业务逻辑。
- UI 代码放在 `ui/` 或 `ui/tabs/`；核心逻辑放在 `core/`；公共工具放在 `utils/`。

### 二、类型安全优先

#### 2.1 使用枚举替代字符串匹配
- 涉及业务模式的参数必须使用枚举类型。
- 后端入口收到前端字符串后，必须第一时间转换为对应 `Enum`。
- 后续流程统一使用 `Enum` 相等判断。
- 禁止字符串包含/相等判断业务分支。
- 示例文件：`agnets/example-01-enum.md`

#### 2.2 使用数据类统一参数管理
- 多参数函数使用 `@dataclass` 封装参数。
- 示例文件：`agnets/example-06-dataclass-request.md`

### 三、模块化拆分原则

#### 3.1 大文件拆分
- 单个文件超过 500 行时，必须评估按功能拆分。
- 拆分策略：按功能模块拆分、按抽象层次拆分（core/ui/utils）。

#### 3.2 提取独立功能模块
- 可复用功能提取到独立模块，避免“巨型文件 + 巨型函数”。
- 示例文件：`agnets/example-05-module-organization.md`

### 四、静态资源外部化

#### 4.1 CSS/JS 外部化
- 禁止在 Python 文件中硬编码大段 CSS/JS/HTML 字符串。

#### 4.2 统一资源管理
- 根据用途选择目录：
  - `ui/static/`: 通过文件路径直接引用（暴露给 Gradio），用于第三方库与样式文件。
  - `ui/assets/`: 通过 `load_asset_text` / `load_template_text` 读取为字符串，用于 JS 片段和模板。

#### 4.3 模板驱动架构
- 复杂UI组件使用模板文件，不使用 Python 字符串拼接 HTML。
- 示例文件：`agnets/example-03-ui-static-assets.md`

### 五、静态文件选择规则

#### 5.1 何时使用 ui/static
- 需要在 HTML 中通过 URL 直接引用的资源。
- 典型：`libs/`、`css/`。

#### 5.2 何时使用 ui/assets
- 需要读取为字符串并在 Python / Gradio 回调中使用的资源。
- 典型：`js/` 代码片段、`template/` 模板。

#### 5.3 资源引用规范
- 所有静态资源统一通过 `ui/assets.py` 暴露与引用。
- 禁止跨层级导入与散落的直接文件读取。

### 六、函数单一职责

#### 6.1 函数拆分原则
- 单个函数超过 100 行或承担多个职责时，必须拆分。
- 协调函数负责校验与分发，专职函数负责具体逻辑。
- 示例文件：`agnets/example-02-single-responsibility.md`

#### 6.2 提取重复逻辑
- 出现 2 次以上相同逻辑，必须提取为独立函数。
- 示例文件：`agnets/example-07-extract-duplicate-logic.md`

### 七、渐进式重构（适用于重构现有代码）

#### 7.1 小步提交原则
- 每次重构只做一件事，保持功能不变。
- 禁止一次提交完成多项不相关重构。

#### 7.2 非公共 API 重构策略
- 本软件不公开对外接口。
- 重构时不需要兼容性包装器，直接修改所有调用处使用新接口。

### 八、代码清理

#### 8.1 移除未使用代码
- 重构后移除不再使用的导入、函数、变量。

#### 8.2 简化导入结构
- 导入顺序固定：标准库 -> 第三方库 -> 本地模块。
- 示例文件：`agnets/example-08-import-order.md`
 
## 规则优先级

**必须遵守**:
1. 分层解耦：core 层不得返回用户可见文本
2. 类型安全：业务模式必须使用枚举，禁止字符串匹配
3. 静态资源：禁止硬编码 CSS/JS/HTML 字符串
4. 国际化：core 层使用 `make_status_tag`，UI 层使用 `resolve_i18n_text`

**强烈推荐（影响代码质量）**:
1. 代码清理：及时移除未使用代码
2. 提取重复：2 次以上相同逻辑必须提取
 
## 执行要求
- 修改后必须运行相关测试（至少覆盖变更路径）。
- 新增 UI 按钮/交互时，必须补对应测试。
- 保持改动最小，不做无关格式化。
